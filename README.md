## 📢 프로젝트 소개

**🧠'어디까지 만들 수 있을까..?'** <br/>
"기능 하나라도 우리 손으로 직접 완성해보자" 라는 마음으로 **브런치스토리**를 구현했습니다.

### 👤 회원 관련 기능 <br/>

## 🐞 트러블 슈팅

<table>
  <thead>
    <tr>
      <th style="width: 100px;"> 👤 </th>
      <th>❗ 문제점 & 해결 사례 🛠️</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>지안</td>
      <td>

 **메인 홈페이지** <br/>
- 요즘 뜨는 브런치 : 타입관련 문제로 인한 필터 처리 실패 (타입변환후 정렬성공)<br/>
- TOP 구독 작가 : 북마크 API와 유저 API의 사용 혼란 (유저API의 <code>bookmarkedBy</code> 속성 사용후 해결)<br/>
- 오늘의 작가 : 작가 소개글이 일정 글자를 넘어갈때 화면이 깨짐(css의 <code>overflow hidden</code>으로 해결), 
책 리스트 박스를 출력시 컴포넌트 깨짐 (컴포넌트 css 반응형으로 수정후 해결)<br/>

**상세페이지** <br/>
- 글 출력 : 게시글 id와 작가 id의 API를 가져오는중 401 에러 (토큰만료로 인한 에러)<br/>
- 작가 소개 출력
타입관련 빌드가 안되는 에러 (타입 변경후 해결)
- 네비게이션 
구독 상태 체크시 api 관련된 혼동 (bookmarks의 user는 구독 post는 좋아요 기능으로 분리)
</td>
    </tr>

  </tbody>
